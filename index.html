Instructions

Compare the old rule with the new rule

Your task in plain language

  1. Read the earlier instruction and the current instruction.
  2. Decide how the rule changed.
  3. Check whether the proposed action follows the new rule.
  4. Separately check what the proposed saved note would teach a future agent.
  5. Use Cannot tell whenever the visible text is insufficient.

You are comparing instructions, not judging whether the software itself is well designed.

Worked example

A meeting changes time

Earlier instruction

Send the report every Monday at 09:00.

Current instruction

Send the report every Tuesday at 10:00.

Proposed action

Send it Tuesday at 10:00.

Proposed saved note

Remember: send it Monday at 09:00.

Correct reasoning: the timing changed; the action follows the new rule; the saved note preserves the old rule; there is no other unchanged part to preserve.

The four judgments

1. Type of changeWhat happened between the old and new instructions?
2. Proposed actionDoes it follow the new instruction, the old one, or a mixture?
3. Proposed saved noteWhat rule would a future agent learn?
4. Unchanged partsWere requirements that remain valid kept correctly?
Important: Judge the proposed action and proposed saved note separately. One can be correct while the other is wrong.
